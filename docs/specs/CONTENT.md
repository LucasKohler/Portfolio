# CONTENT.md

## Content Strategy

The portfolio copy must be professional, direct and technically credible.

The writing should sell Lucas as a Software Engineer without sounding exaggerated, generic or artificial.

---

## Tone

Use language that is:

- Clear
- Technical
- Objective
- Mature
- Business-oriented
- Recruiter-friendly

Avoid language that is:

- Overly casual
- Excessively emotional
- Generic
- Inflated
- Cliché
- Fake

---

## Avoid These Phrases

Do not use:

- Passionate developer
- Technology lover
- Always learning
- Ninja
- Rock star
- Guru
- Wizard
- I love technology
- Dreaming in code
- Pixel-perfect craftsman
- 10x developer

---

## Preferred Themes

Use themes like:

- Software engineering
- Business impact
- Maintainability
- Architecture
- Performance
- Scalable systems
- Data-driven systems
- Clean component organization
- Dockerized development
- API design
- AI-assisted workflows
- Engineering judgment


Overview:

```txt
This project is the foundation of my public engineering portfolio. It is designed to present my profile, skills, technical experiments and documented projects in a clear, scalable and recruiter-friendly way.
```

Purpose:

```txt
The goal is to create a polished web experience that works both as a professional presentation and as a technical playground for modern frontend architecture, .NET API design, Dockerized development, routing, design systems and AI-assisted workflows.
```

Technical Highlights:

```txt
Next.js App Router frontend
.NET backend API for project data
Dockerized local development
SPA-style navigation and dynamic project routing
Reusable project card architecture
Scalable project data structure
Responsive interface design
Clean component organization
Professional copywriting and technical positioning
```

Implementation Notes:

```txt
Projects should be rendered dynamically from backend-provided data, allowing new projects to be added without redesigning the page. Each project card routes to a dynamic detail page using a slug-based route such as /projects/portfolio-platform.
```
